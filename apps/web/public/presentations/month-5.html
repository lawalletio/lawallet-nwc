App 1 · Card Installer

Operators write blank NTAG424s
in seven taps.

A new native Android app at lawalletio/card-installer — forked from bolt-nfc-android-app. The operator pairs once with a QR, then every blank card flows through the same seven-step ritual.

Step 01

Pair the device

Operator scans the QR shown in admin web → JWT in 60s.

Step 02

Tap a blank card

App reads the NTAG424 UID over Android NFC.

Step 03

Pick a design

Picker is populated from GET /api/card-designs.

Step 04

Backend issues keys

Random k0–k4 generated, status → KEYS_GENERATED.

Step 05

Write to chip

App writes k0–k4 + LNURLw URL via Android NFC.

Step 06 — 07

Confirm written

State flips to WRITTEN. OTC is live.

Backend bones · 1 of 2

QR JWT pairing for clients
that don't speak Nostr.

Native Android apps can't sign NIP-98 — they have no nsec. So we bridge: the admin web (which IS logged in via NIP-98) generates a 60-second pairing code; the Android app scans the QR and exchanges the code for a long-lived operator JWT.

POST /api/jwt/qr-pair-init
Auth required · operator only

Generates a 128-bit URL-safe random pairing code, stores it in-memory with a 60-second TTL alongside the operator's role + permissions. Returns { pairingCode, expiresAt }.

POST /api/jwt/qr-pair-claim
Public · rate limited 5/min/IP

Android sends the scanned pairing code. Server validates (exists, not expired, not used), marks it used, and emits a 24-hour JWT with the operator's role and perms.

Backend bones · 2 of 2

A card has five states.
Every transition is guarded.

A single helper at lib/cards/state-machine.ts validates every transition. Out-of-order writes return 409 Conflict. Admin UI shows a status badge per card. The Manager app reflects state changes live via SSE.
