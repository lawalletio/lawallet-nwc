## Overview

The main application in `lawallet-nwc` serves the frontend, REST API, and LUD-16 lightning address resolution.

**Container**: `lawallet-web`
**Port**: 3000
**Storage**: Own PostgreSQL database (Prisma ORM)

---

## Responsibilities

- Serve frontend (React + Tailwind/shadcn)
- Expose REST API for cards, card designs, users, settings, JWT auth, and remote connections
- Resolve lightning addresses via `/api/lud16/[username]`
- Optionally create Alby Hub sub-accounts during user creation
- Serve placeholder landing, admin, and wallet pages

Not implemented in the current repo:

- `.well-known/lnurlp` and `.well-known/nostr.json`
- Redirect-based address resolution
- Webhook management and payment history APIs
- A completed user dashboard UI

## Key Endpoints

### Authentication

- `POST /api/jwt` ŌĆö Exchange a NIP-98 request for a JWT
- `GET /api/jwt` ŌĆö Validate an existing JWT
- `GET /api/jwt/protected` ŌĆö Example protected JWT route
- `POST /api/jwt/protected` ŌĆö Example protected JWT route

### Users and Addresses

- `GET /api/users/me` ŌĆö Load or create the current user
- `PUT /api/users/:id/lightning-address` ŌĆö Assign or replace a username
- `PUT /api/users/:id/nwc` ŌĆö Store an NWC URI for the user
- `GET /api/lightning-addresses` ŌĆö Admin list of lightning addresses
- `GET /api/lightning-addresses/counts` ŌĆö Admin counts
- `GET /api/lightning-addresses/relays` ŌĆö Relay information
- `GET /api/lud16/:username` ŌĆö LUD-16 payRequest for NWC-backed users
- `GET /api/lud16/:username/cb` ŌĆö LUD-16 callback

### Cards and Card Designs

- `GET /api/cards` and `POST /api/cards` ŌĆö Card listing and creation
- `GET /api/cards/:id` and `DELETE /api/cards/:id` ŌĆö Card detail and deletion
- `GET /api/cards/:id/write` ŌĆö Card write payload
- `GET /api/cards/:id/scan` ŌĆö LNURL-withdraw style card scan response
- `GET /api/cards/:id/scan/cb` ŌĆö Card scan callback
- `GET /api/cards/otc/:otc` ŌĆö OTC lookup
- `POST /api/cards/otc/:otc/activate` ŌĆö OTC activation
- `GET /api/card-designs/list` ŌĆö Card designs
- `GET /api/card-designs/count` ŌĆö Card-design count
- `GET /api/card-designs/get/:id` ŌĆö Card-design detail
- `POST /api/card-designs/import` ŌĆö Import card designs

### Admin and Device Flows

- `GET /api/settings` and `POST /api/settings` ŌĆö Settings management
- `GET /api/admin/assign` and `POST /api/admin/assign` ŌĆö Admin assignment
- `GET /api/root/assign` and `POST /api/root/assign` ŌĆö Root assignment
- `GET /api/remote-connections/:externalDeviceKey` ŌĆö Remote connection bootstrap
- `POST /api/remote-connections/:externalDeviceKey/cards` ŌĆö Remote card initialization
