## What the listener does

The **NWC Listener** (`apps/listener`) is an optional companion service that keeps
persistent Nostr relay connections open for every active NWC wallet on your
instance:

- **Incoming payments become webhooks** ‚Äî NIP-47 `payment_received` /
  `payment_sent` notifications are forwarded to the web app (HMAC-signed),
  flipping invoices to paid in real time.
- **Card payments get fast without duplicate risk** ‚Äî an idempotent,
  wallet-ID-routed endpoint reuses the already-open relay socket, journals the
  dispatch before NWC, and reconciles late results by request ID.
- **Other outgoing NWC calls get fast** ‚Äî the general compatibility proxy can
  reuse the pool for balance checks and invoice minting.
- **Live dashboard** ‚Äî `Admin ‚Üí NWC Listener` shows active relays, connections
  and recent events.

LaWallet works without it: NWC calls fall back to per-request relay
connections, and no incoming payment notifications are received.

## Do I need to deploy anything?

| Your deployment      | Listener setup                                                                                                                                                                           |
| -------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Docker Compose**   | Optional ‚Äî enable the `listener` profile and provide a generated secret. Plain `docker compose up` remains web-only.                                                                       |
| **Umbrel / Start9**  | Nothing ‚Äî bundled with the package since v1.1.0.                                                                                                                                         |
| **Vercel / Netlify** | Yes ‚Äî serverless platforms can't run a long-lived websocket process. Host the listener elsewhere (below), then connect it in **Settings ‚Üí NWC Services**.                                |
| **Local dev**        | `pnpm dev:listener` (env is provisioned by `pnpm dev:setup`).                                                                                                                            |

## Environment variables

The listener is one small container ‚Äî use the prebuilt multi-arch image
[`masize/lawallet-nwc-listener`](https://hub.docker.com/r/masize/lawallet-nwc-listener)
(published alongside each release) or build from
`apps/listener/Dockerfile`. Default port 4100. It needs:

| Variable                        | Required | Description                                                                                                                                                                                                          |
| ------------------------------- | -------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `DATABASE_URL`                  | ‚úÖ       | The **same Postgres** your web app uses (it reads NWC wallets and stores processed events plus the idempotent payment journal).                                                                                      |
| `LISTENER_AUTH_SECRET`          | ‚úÖ       | Webhook HMAC secret, min 32 chars. Also guards the listener API when no dedicated request secret is set. Must match the web-side webhook secret.                                                                     |
| `LISTENER_REQUEST_AUTH_SECRET`  | ‚Äî        | Optional dedicated web‚Üílistener bearer secret, min 32 chars. Set the same value on web; otherwise both sides fall back to `LISTENER_AUTH_SECRET`.                                                                    |
| `WEB_ORIGIN`                    | ‚úÖ       | Public URL of your web app (e.g. `https://wallet.example.com`) ‚Äî payment webhooks are POSTed back there.                                                                                                             |
| `LISTENER_PORT`                 | ‚Äî        | HTTP port (default `4100`).                                                                                                                                                                                          |
| `LOG_LEVEL` / `LOG_PRETTY`      | ‚Äî        | Logging (default `info` / JSON).                                                                                                                                                                                     |
| `EVENT_RETENTION_DAYS`          | ‚Äî        | Days of processed events kept for dedup + dashboard feed (default 30).                                                                                                                                               |
| `NWC_REQUEST_TIMEOUT_MS`        | ‚Äî        | Default timeout for proxied `/nwc/request` calls (default 30000).                                                                                                                                                    |
| `WEBHOOK_MAX_ATTEMPTS`          | ‚Äî        | Inline webhook delivery attempts before the sweep takes over (default 5).                                                                                                                                            |
| `RECONCILE_INTERVAL_MS`         | ‚Äî        | Periodic full pool reconcile, the LISTEN/NOTIFY safety net (default 300000).                                                                                                                                         |
| `CATCHUP_ENABLED`               | ‚Äî        | Missed-event recovery after downtime (default `true`).                                                                                                                                                               |
| `CATCHUP_MAX_WINDOW_HOURS`      | ‚Äî        | Furthest back a catch-up looks (default 24).                                                                                                                                                                         |
| `CATCHUP_OVERLAP_SECONDS`       | ‚Äî        | Overlap subtracted from the per-wallet cursor (default 300).                                                                                                                                                         |
| `CATCHUP_INTERVAL_MS`           | ‚Äî        | Periodic safety catch-up; 0 disables (default 900000).                                                                                                                                                               |
| `DEAD_WALLET_DETECTION_ENABLED` | ‚Äî        | Auto-archive a disposable LNCurl wallet as `DEAD` after it stops answering for `DEAD_THRESHOLD_HOURS` while its relays stay up (default `true`; set `false` to disable). Non-LNCurl wallets are never auto-archived. |
| `DEAD_THRESHOLD_HOURS`          | ‚Äî        | Silence (relays up) before a wallet is declared dead (default 4).                                                                                                                                                    |

**Connectivity requirements:** the listener must reach your Postgres and the
Nostr relays; your web app must reach the listener's URL; the listener must
reach your web app's URL. With a hosted Postgres (Neon, Supabase, Railway
Postgres ‚Äî typical for Vercel deployments) all of this works over the public
internet.

## Hosting options for Vercel/Netlify deployments

### Railway (walkthrough)

1. In [Railway](https://railway.com), create a **New Project ‚Üí Deploy from
   GitHub repo** and pick your fork of `lawallet-nwc` (or the upstream repo).
2. In the service settings, set **Dockerfile Path** to
   `apps/listener/Dockerfile` (Root Directory stays the repo root ‚Äî the build
   needs the whole workspace).
3. Add the environment variables from the table above. Generate the
   `LISTENER_AUTH_SECRET` in **Settings ‚Üí NWC Services** (the Generate button)
   and copy the same value here. For separately managed HTTP credentials, set
   the same `LISTENER_REQUEST_AUTH_SECRET` on both hosts; this is optional.
4. Under **Networking**, generate a public domain for the service.
5. Deploy. The healthcheck path is `/health`.

### Render

Create a **Web Service** from the repo, environment **Docker**, set the
Dockerfile path to `apps/listener/Dockerfile`, add the env vars, and use the
assigned `onrender.com` URL.

### Fly.io

`fly launch --dockerfile apps/listener/Dockerfile --no-deploy`, add the env
vars with `fly secrets set`, then `fly deploy`. Expose port 4100 in the
generated `fly.toml`.

### Any VPS with Docker

```bash
docker run -d --restart unless-stopped -p 4100:4100 \
  -e DATABASE_URL="postgresql://..." \
  -e LISTENER_AUTH_SECRET="<same secret as the web app>" \
  -e WEB_ORIGIN="https://wallet.example.com" \
  masize/lawallet-nwc-listener:latest
```

To separate HTTP and webhook credentials, also pass
`-e LISTENER_REQUEST_AUTH_SECRET="‚Ä¶"` and configure the same value on web.

Put it behind your reverse proxy with TLS if the web app connects over the
public internet.

## Connecting it to LaWallet

1. Open **Admin ‚Üí Settings ‚Üí NWC Services**.
2. Paste the listener's public URL (e.g.
   `https://lawallet-listener.up.railway.app`).
3. Set the **shared secret** ‚Äî use **Generate**, then copy the same value to
   the listener host's `LISTENER_AUTH_SECRET` (redeploy it if needed). The
   Settings field is the compatibility shared credential; dedicated
   `LISTENER_REQUEST_AUTH_SECRET` separation is available for env-managed
   deployments.
4. Click **Test connection** ‚Äî you should see uptime and connection counts.
5. Flip **Enable listener integration**. The **NWC Listener** dashboard
   appears in the sidebar.

For the bundled private listener, generate a secret and opt in explicitly:

```bash
LISTENER_URL=http://listener:4100 \
LISTENER_AUTH_SECRET="$(openssl rand -hex 32)" \
docker compose --profile listener up -d
```

Persist those values in `.env` for subsequent Compose commands. Plain
`docker compose up -d` leaves the listener unconfigured and all payments use
the regular direct NWC path.

## Health & failure modes

- `GET /health` ‚Äî liveness (used by container healthchecks).
- `GET /ready` ‚Äî unauthenticated capability/readiness probe. Confirm it lists
  `"nwc_payments_v1"` and shows ready/not-ready wallet counts.
- `GET /status` ‚Äî authenticated status (relays, connections, counters) ‚Äî
  surfaced in the admin dashboard.
- Wallet state normally progresses `connecting ‚Üí negotiating ‚Üí ready`.
  `disconnected` means the relay link dropped and is reconnecting; `error`
  means warm-up failed and is backing off. A wallet is not used for the card
  fast path until it reaches `ready`.
- **Listener down** ‚Üí the web app keeps working, degraded to per-request
  relay connections for payments that have not been sent to listener; the
  dashboard shows _unreachable_. If a payment POST times out or resets, web
  reconciles that request ID and does not issue another NWC payment.
- **Web app down** ‚Üí events accumulate in the listener's store and are
  re-delivered by a sweep when it returns.
