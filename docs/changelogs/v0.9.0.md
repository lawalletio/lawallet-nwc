# v0.9.0

## Release date

2026-04-05

## Summary

This release promotes the full front-end/admin refresh from PR #186 and formalizes the split between the NWC app and the public marketing site.

## Highlights

### App & onboarding

- New landing-to-app claim flow with debounced username availability validation
- Authenticated users are redirected straight into the dashboard flow
- Settings page now includes save/cancel/revert actions and unsaved-change protection
- Shared NostrConnect form extracted for cleaner auth UX

### UI refresh

- Updated admin/sidebar behavior and responsive navigation fixes
- Theme-aware CTA styling and assorted polish across dashboard flows
- Dependency audit cleanup and auth/lint fixes from the final PR 186 merge

### Landing migration

- The public landing is now owned by the dedicated [`lawallet-landing`](https://github.com/lawalletio/lawallet-landing) repository
- `lawallet-nwc` now treats `/` as a redirect to the landing site (`NEXT_PUBLIC_LAWALLET_LANDING_URL`, default `https://lawallet.io`)
- Product traffic should use `/admin` for the app entrypoint

## Included work

- [PR #186](https://github.com/lawalletio/lawallet-nwc/pull/186) — front-end refactor and landing/app flow improvements
- Root redirect update for the landing split
- Version bump to `0.9.0`

## Upgrade notes

- If you deploy `lawallet-nwc` behind the public domain, set `NEXT_PUBLIC_LAWALLET_LANDING_URL` when the landing is hosted elsewhere
- If you want app-first behavior, link users directly to `/admin`
