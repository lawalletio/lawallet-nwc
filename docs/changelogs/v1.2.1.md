# v1.2.1

## Release date

2026-07-07

## Summary

Payment webhooks are now **never dropped**: the listener replaces its hard
25-attempt sweep cap with indefinite retry paced by a per-event backoff column,
so a temporarily unreachable web app no longer permanently loses events. The
listener dashboard also gains live realtime status without feed flicker, and
admins can view any Lightning address read-only.

## Highlights

### Listener (#82, #84)

- **Never drop payment webhooks** — remove the 25-attempt cap; retry
  indefinitely on a per-event backoff (2 min → 1 h) with backlog logging (#84)
- **Live realtime status** + stop the events feed flickering (#82)

### Addresses (#83)

- Admins can **view any Lightning address read-only** (#83)
