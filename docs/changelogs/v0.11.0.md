# v0.11.0

## Release date

2026-06-10

## Summary

Two releases in one: the **Remote Wallets platform** (the new `RemoteWallet`
model with a pluggable driver layer, Connection Map UI, and QR device-token
login) and the **DevX program** — a ground-up overhaul of how this repo is
developed: one-command setup, isolated parallel worktrees, a rewritten Turbo
graph with remote caching, a real Playwright E2E tier, a docs-drift CI gate,
a debugging toolkit, and the headline feature for forks: an **in-codebase
plugin system** that adds pages, API routes, and lifecycle hooks without
touching core. Releases themselves are now cut by a one-dispatch workflow —
this is the first version shipped through it.

## Highlights

### Remote Wallets epic (#267, #270, #271, #274, #277, #279, #280)

- New `RemoteWallet` model; legacy `NWCConnection` rows migrated forward
- **`RemoteWalletDriver` interface + NWC driver** — the pluggable pattern
  (module-level registry, per-driver Zod config) that now anchors all
  integrations
- User wallet UI with full CRUD; Cards + LightningAddresses bind through
  RemoteWallet
- **Connection Map** — desktop canvas + mobile tabbed lists visualizing
  address/card → wallet bindings

### Device tokens & auth (#281, #282, #2, #3, #7)

- **QR-based JWT device-token login** (M5 B.0), tokens scoped to the issuing
  instance via the `apiUrl` claim; 30-day expiry cap removed; claims schema
  documented

### In-codebase plugin system (#18, #24)

- Extend the platform — admin pages, sidebar nav (RBAC-gated), API routes
  under `/api/plugins/<id>/…`, lifecycle hooks (`invoice:paid`) — with **one
  directory plus one registration line; zero core edits, zero migrations**
  (JSON-first `PluginRecord` storage)
- Badges reference plugin, `pnpm plugin:new` scaffold, and the rulebook in
  `docs/PLUGINS.md`

### Developer experience program (#8, #11, #12, #13, #21, #22, #23, #16, #20, #25)

- **One-command onboarding**: `pnpm start:dev-server` — deps, env files,
  isolated per-checkout Postgres, migrate + seed, dev server; re-runs never
  wipe data; every git worktree self-provisions
- **Turbo graph rewrite**: quality tasks no longer wait on builds, Prisma
  client generation is cached — typecheck 11s → 56ms, build 27s → 71ms on
  cache hits; CI runs four parallel peers with **Vercel Remote Cache**
- **Playwright E2E tier** against a real server + dedicated `*_e2e` database
  with JWT-minting auth fixtures, plus a `vitest bench` harness
- **Docs drift gate**: `pnpm docs:check` fails CI when an API route lacks an
  OpenAPI operation (ratchet allowlist), an MDX page is orphaned, or the
  committed spec snapshot is stale; remote-wallets fully documented (#19)
- **Debugging toolkit**: slow-query warn logging, timing spans on NWC ops,
  `pnpm logs --reqId=…` request tracing, and the `docs/DEBUGGING.md` runbook
- Claude Code layer: scoped CLAUDE.md files, 8 subagents, 8 slash commands,
  format-on-edit hook
- `.env.example` now documents every validated variable (and the
  `PRETTY_LOG` → `LOG_PRETTY` doc bug is fixed)

### Admin & platform polish (#4, #5, #6, #9, #10, #229, #273, #276, #278)

- **Domain onboarding wizard** with persisted verification state
- veintiuno.lat card-design catalog import; searchable UserSelector with
  avatars; default Blossom servers; local-development banner; amber login

### Infrastructure & analytics (#201, #202, #204, #206, #207, #262, #268)

- Standalone `lawallet` installer CLI (curl|bash bootstrap)
- Anonymous Google Analytics across web + docs; Codecov wiring
- Next.js bumped to ^16.2.6 (6 high-severity advisories patched)

### Release flow (#26)

- `pnpm release:patch|minor|major` → gates, lockstep version bump
  (root + web + cli), changelog scaffold, tag, GitHub Release — v0.11.0 is
  the first release cut this way
